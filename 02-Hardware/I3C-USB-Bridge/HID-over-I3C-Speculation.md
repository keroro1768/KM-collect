# HID Over I3C 推測規格

> 本文件為基於 Microsoft HID over I2C 規範與 MIPI I3C 特性進行的推測
> 僅供參考，實際規格需依據未來發布的官方文件

---

## 1. 概述

### 1.1 背景

HID over I3C 是將 HID (Human Interface Device) 協定運行在 MIPI I3C 匯流排上的推測規範。基於 HID over I2C 的架構，配合 I3C 的新特性進行擴展。

### 1.2 設計目標

| 目標 | 說明 |
|------|------|
| **相容性** | 維持與 HID over I2C 相似的應用層結構 |
| **高效能** | 利用 I3C 的高速傳輸能力 |
| **低功耗** | 繼承 I3C 的低功耗特性 |
| **熱插拔** | 支援 I3C 的 Hot-Join 機制 |

---

## 2. I3C 與 I2C 的差異對照

### 2.1 匯流排特性比較

| 特性 | I2C | I3C | 對 HID over 的影響 |
|------|-----|-----|-------------------|
| **速度** | 400 Kbps | 12.5 Mbps (SDR) | 傳輸效率提升 ~30x |
| **通訊方式** | 半雙工 | 半雙工 (SDR) / 全雙工 (HDR) | 類似但支援 HDR |
| **位址分配** | 固定 | 動態分配 | 需支援 Dynamic Address |
| **中斷機制** | 外部中斷 | In-Band Interrupt (IBI) | 支援 IBI 通知 |
| **功耗** | 較高 | 較低 | 更適合行動裝置 |
| **從屬數量** | 有限 | 最多 80+ 裝置 | 可支援更多 HID 裝置 |

### 2.2 推測的傳輸效率

```
I2C HID: ~400 Kbps → 實際傳輸 ~200-300 Kbps
I3C HID (SDR): 12.5 Mbps → 實際傳輸 ~8-10 Mbps
I3C HID (HDR-DDR): 25 Mbps → 實際傳輸 ~20 Mbps
```

### 3.1 層級架構

```
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    I3C Host Controller                      │
│  ┌──────────────┐    ┌──────────────┐                   │
│  │   I3C HCI   │◄───│  HID Parser │                   │
│  └──────────────┘    └──────────────┘                   │

                              │

└─────────────────────────────────────────────────────────────┘
```

### 3.2 推測的暫存器映射

基於 HID over I2C，推測的 I3C 暫存器：

| 暫存器 | 位址 | 說明 |
|--------|------|------|
| COMMAND_REG | 0x00 | 命令暫存器 |
| DATA_REG | 0x04 | 資料傳輸暫存器 |
| STATUS_REG | 0x08 | 狀態暫存器 |
| INT_REG | 0x0C | 中斷暫存器 (支援 IBI) |
| DESC_ADDR_REG | 0x10 | Descriptor 位址 |
| REPORT_ADDR_REG | 0x14 | Report 緩衝區位址 |

## 4. 通訊協定

### 4.1 I3C 位址分配

| 項目 | 推測值 |
|------|--------|
| HID 裝置預設位址 | 0x08 - 0x0F (可設定) |
| 動態位址範圍 | 0x08 - 0x77 |
| Broadcast 位址 | 0x7E |

### 4.2 命令格式

```
┌─────────────────────────────────────────────────────────────┐
│ Byte 0  │ Byte 1   │ Byte 2-3    │ Byte 4-n            │
├──────────┼───────────┼──────────────┼─────────────────────┤
│  Destination │ Command │ Length     │ Data                │
│  Address    │ Code    │ (16-bit)   │                     │
│  (7-bit)    │ (8-bit) │            │                     │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 HID 命令碼 (推測)

| 命令碼 | 名稱 | 說明 |
|--------|------|------|
| 0x01 | GET_DESCRIPTOR | 取得 HID Descriptor |
| 0x02 | GET_REPORT_DESC | 取得 Report Descriptor |
| 0x03 | SET_REPORT | 設定 Report |
| 0x04 | GET_REPORT | 取得 Report |
| 0x05 | SET_IDLE | 設定閒置狀態 |
| 0x06 | GET_IDLE | 取得閒置狀態 |
| 0x07 | SET_POWER | 設定電源狀態 |
| 0x08 | GET_POWER | 取得電源狀態 |

## 5. 描述符格式

### 5.1 HID Descriptor (推測)

```c
typedef struct {
    uint16_t wHIDDescLength;    // Descriptor 長度
    uint16_t bcdVersion;        // HID 版本 (0100 = 1.0)
    uint8_t  bCountryCode;       // 國家碼
    uint8_t  bNumDescriptors;    // 子描述符數量
    uint8_t  bDescriptorType;    // Report Descriptor Type (0x22)
    uint16_t wDescriptorLength;  // Report Descriptor 長度
} HID_I3C_Descriptor;
```

### 5.2 Report Descriptor

與 HID over I2C 相同，使用標準 HID Report Descriptor 格式。

## 6. 報告傳輸

### 6.1 Input Reports

| 傳輸方式 | I2C | I3C (推測) |
|----------|-----|------------|
| Polling | 主動輪詢 | 主動輪詢 |
| IBI | 不支援 | **支援** ← 差異點 |
| DMA | 有限 | 完整支援 |

### 6.2 In-Band Interrupt (推測)

I3C 的 IBI 機制可讓 HID 裝置主動通知主機：

```
I3C Bus:
  [HEADER: IBI + Target Address] → [IBI Header] → [Input Report Data]
```

**優點：**
- 更低的延遲
- 無需額外中斷線
- 支援多裝置同時通知

## 7. 與 I2C 版本的主要差異

### 7.1 功能增強

| 功能 | HID over I2C | HID over I3C (推測) |
|------|--------------|---------------------|
| **傳輸速度** | 400 Kbps | 12.5 Mbps (SDR) / 25 Mbps (HDR) |
| **中斷機制** | 外部中斷 | IBI (In-Band Interrupt) |
| **熱插拔** | 有限支援 | 完整支援 (Hot-Join) |
| **多裝置** | 需要額外仲裁 | 內建多主控支援 |
| **功耗** | 標準 | 更低 |

### 7.2 向後相容性

```
┌─────────────────────────────────────────────────────────────┐
│                      I3C Bus                               │
│  ┌──────────────┐    ┌──────────────┐    ┌────────────┐ │
│  │ HID over I3C│    │ HID over I2C  │    │  Legacy    │ │
│  │ Device (新)  │    │ Device (橋接) │    │  I2C Device│ │
│  └──────────────┘    └──────────────┘    └────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

I3C 完全向後相容 I2C，可透過橋接晶片支援傳統 I2C HID 裝置。

## 9. 總結：推測的可行性

### 9.1 技術可行性

| 項目 | 評估 |
|------|------|
| **底層協定的相似度** | 高 (80-90%) |
| **實作複雜度** | 中等 |
| **需要新硬體** | 需要 I3C 相容控制器 |
| **軟體相容性** | 高 (可複用大部 分驅動程式) |

### 9.2 關鍵差異點

1. **IBI 機制** - I3C 獨有的 In-Band Interrupt
2. **動態位址** - I3C 支援動態位址分配
3. **HDR 模式** - 可支援更高的資料率
4. **Hot-Join** - 更好的熱插拔支援

### 9.3 與 USB HID 的比較

| 特性 | USB HID | HID over I3C (推測) |
|------|---------|---------------------|
| 速度 | 12-480 Mbps | 12.5-25 Mbps |
| 功耗 | 較高 | 較低 |
| 複雜度 | 中等 | 簡單 |
| 熱插拔 | 支援 | 支援 |
| 驅動程式 | 內建 | 需開發 |
