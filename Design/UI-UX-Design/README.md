## 3. UI-UX-Principles

UI/UX 設計核心原則

### 十大可用性啟發 (Nielsen)

1. **系統狀態可見性** - 讓使用者始終知道發生什麼
2. **系統與現實匹配** - 使用使用者語言
3. **使用者控制與自由** - 支援 undo/redo
4. **一致性与標準** - 遵循平台慣例
5. **錯誤預防** - 預設比錯誤訊息更好
6. **識別而非記憶** - 減少記憶負擔
7. **彈性與效率** - 為新手和專家提供不同路徑
8. **美觀與簡約** - 去除不相關資訊
9. **錯誤恢復** - 清楚的錯誤訊息與解決方案
10. **幫助與文檔** - 易於搜尋的幫助

### 參考資源

- [NN/g - 10 Usability Heuristics](https://www.nngroup.com/articles/ten-usability-heuristics/)
- [Figma - 7 Key UI Design Principles](https://www.figma.com/resource-library/ui-design-principles/)

## 4. Design-Patterns

設計模式與範例

### 常見模式

- **Ribbon Interface** - 功能區域帶狀工具列
- **Dock Panels** - 可停靠面板
- **Master-Detail** - 主從視圖
- **Wizard** - 向導式步驟
- **Modal Dialog** - 模態對話框

### 桌面應用獨有模式

- **Small point sizes** - 標籤使用小字體
- **Dense arrays** - 密集的功能表和選項
- **Detachable panels** - 可分離面板
- **Contextual information** - 上下文資訊顯示
